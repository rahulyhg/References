Sign In with Phone Number

Log in and sign up with phone number.
Step 1 of 3

Add a Sign In button

You can add a button to let your users authenticate themselves with their phone number. Add the following to your layout where you’d like the button to appear.

activity_main.xmlObjective CSwift

<RelativeLayout xmlns:android="http://schemas.android.com/apk/res/android"
    xmlns:tools="http://schemas.android.com/tools" android:layout_width="match_parent"
    android:layout_height="match_parent" android:paddingLeft="@dimen/activity_horizontal_margin"
    android:paddingRight="@dimen/activity_horizontal_margin"
    android:paddingTop="@dimen/activity_vertical_margin"
    android:paddingBottom="@dimen/activity_vertical_margin" tools:context=".MainActivity">


    <com.digits.sdk.android.DigitsAuthButton
        android:id="@+id/auth_button"
        android:layout_width="wrap_content"
        android:layout_height="wrap_content" />


</RelativeLayout>
Step 2 of 3

Register an AuthCallback

Add a Sign In with Digits button to allow users to register and sign in to your app using their phone number. In this example, we trigger a toast after successfully authenticating the user.

MainActivity.javaObjective CSwift

import com.digits.sdk.android.*;
public class MainActivity extends Activity {

  @Override
  protected void onCreate(Bundle savedInstanceState) {
    super.onCreate(savedInstanceState);
    setContentView(R.layout.activity_main);


    DigitsAuthButton digitsButton = (DigitsAuthButton) findViewById(R.id.auth_button);
    digitsButton.setCallback(new AuthCallback() {
      @Override
      public void success(DigitsSession session, String phoneNumber) {
        // TODO: associate the session userID with your user model
        Toast.makeText(getApplicationContext(), "Authentication successful for "
              + phoneNumber, Toast.LENGTH_LONG).show();
      }

      @Override
      public void failure(DigitsException exception) {
        Log.d("Digits", "Sign in with Digits failure", exception);
      }
    });

  }

}
Step 3 of 3

Try it out!

Run your app. You should see a “Use my phone number” button. Pressing it will take you into the Digits sign-in flow, which will ask for a country code and phone number.

If you fill out your phone number, you will receive an SMS with a confirmation code to finish logging in.

Completing the flow will show a toast displaying the phone number you signed in with.

Not seeing a successful login?

  • Try again, and verify that you used the correct country code for your phone number
  • Check the logs for any error messages that were printed

You can also get help in the Digits Developer forum.

Theme Your Digits App

Customize Digits to use your own fonts and colors.
Step 1 of 3

Create a Digits theme

Create a custom theme that can be assigned to the Digits authentication screens.

themes.xmlObjective CSwift

<resources>

    <style name="CustomDigitsTheme" parent="android:Theme.Light">
        <item name="android:textColorPrimary">@android:color/black</item>
        <item name="android:textColorSecondary">@android:color/darker_gray</item>
        <item name="android:windowBackground">@android:color/darker_gray</item>
        <item name="android:textColorLink">#ff398622</item>
        <item name="dgts__accentColor">#ffacee</item>
    </style>

</resources>
Step 2 of 3

Set the Digits theme

To use this code you’ll need to already have a DigitsAuthButton in your app. Go through the “Sign In with Phone Number” tutorial if you need one.

Adding a theme to DigitsAuthButton will affect the entire sign in flow. You can change fonts, colors, and add your app’s logo.

MainActivity.javaObjective CSwift

import com.digits.sdk.android.*;

public class MainActivity extends Activity {

    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        TwitterAuthConfig authConfig = new TwitterAuthConfig(TWITTER_KEY, TWITTER_SECRET);


        Digits.Builder digitsBuilder = new Digits.Builder().withTheme(R.style.CustomDigitsTheme); 
        Fabric.with(this, new TwitterCore(authConfig), digitsBuilder.build());

        setContentView(R.layout.activity_main);
        DigitsAuthButton digitsButton = (DigitsAuthButton) findViewById(R.id.auth_button);
        digitsButton.setCallback(new AuthCallback() {
            // …
        });
    }

}
Step 3 of 3

Try it out!

Run the app. You should see that after tapping on the "Use my phone number" button, your theme is applied to the Digits authentication screens.

Not seeing the new theme?

If you're still logged in from a previous Digits session, you won’t see the login flow when you tap the button. Use Digits.clearActiveSession(); in your onCreate, or delete the app from the device to reset the session.

You can also get help in the Digits Developer forum.
